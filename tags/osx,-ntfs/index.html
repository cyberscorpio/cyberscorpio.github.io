ntfs-3g

安装 NTFS-3G

因为我的移动硬盘都当初用 windows 时遗留下来的 NTFS 系统，而 OSX 对 NTFS 只能读不能写，给生活和工作确实带来了很多不便，所以很长时间都想让 OSX 把 NTFS 的读写给支持起来。

但是……对于非 windows 系统写 NTFS 格式，江湖上一直有很多可怕的传说，让我迟迟不敢动手。

前几天终于想开了，与其让自己不方便，不如冒一点风险。于是选择了免费开源的 ntfs-3g，用 brew 安装，但当时心急火燎的，根本就没有看安装说明，不知道还有一些额外的设置，所以装是装上了，却用不起来，于是火大卸之。

今天终于想要彻底解决这个问题。于是仔细研读了一下相关的文档，发现其实还是很容易的。

首先，用 brew update 和 upgrade 升级到最新的 formula。

因为我之前安装的 ntfs-3g 已经被卸载了，所以清理的这一步就不需要了。但如果以后要更新的话，可能还是需要先清理一下，所以还是记录在这里。

注意 ntfs-3g 依赖于 osxfuse 这个模块，而后者实际上就是文件系统，所以我们的清理需要好两个步骤。

清理系统

  • 检查系统中是否有 osxfusefs 模块被加载

    mount -t osxfusefs

如果我们发现有 osxfusefs 的文件系统，那么需要用如下的命令来卸载它们

sudo kextunload -b com.github.osxfuse.filesystems.osxfusefs
  • 卸载 ntfs-3g
brew remove ntfs-3g
brew remove osxfuse

安装 ntfs-3g

好了，我们已经把旧版本的东西清理出去了，现在就可以开始安装 ntfs-3g 了。

brew install ntfs-3g

homebrew 会自动安装依赖的模块（osxfuse）。

如果一切正常，安装完毕之后，我们还需要做一些额外的操作，否则 ntfs-3g 是不能工作的。这一点很重要。

  • 备份和替换 mount_fs 文件
sudo mv /sbin/mount_ntfs /sbin/mount_ntfs.origin
sudo ln -s /usr/local/Cellar/ntfs-3g/2014.2.15/sbin/mount_ntfs /sbin/mount_ntfs
  • 安装 osxfusefs 模块 (可以用 brew info osxfuse 查看)
sudo /bin/cp -RfX /usr/local/opt/osxfuse/Library/Filesystems/osxfusefs.fs /Library/Filesystems
sudo chmod +s /Library/Filesystems/osxfusefs.fs/Support/load_osxfusefs

好了，现在插上 utfs 格式的移动硬盘，如果没有意外的话，硬盘应该可以自动挂载上了，并且可读可写啦。
